MYCELIUM-FX ／ 敘事效果庫

讀者不是讀到世界在劣化，是他的畫面在劣化。

六個捲動驅動的敘事原語，寫文章的時候只標一個 data-fx 屬性。零依賴、沒有 build 步驟，複製兩個檔案就能用在任何部落格。

你正在看的這一頁就在用它——剛才那一下展開不是動畫效果，是 eyelid。

打開示範頁，自己捲一次

六段都能即時拉滑桿調參數。手感這種東西，看說明是判斷不出來的。

六個原語

drag阻力

<div data-fx="drag" data-fx-factor="1.8">

捲動變重。倍率上限是 3——再高就不是「重」，是「壞掉」。

freeze凍結

<p data-fx="freeze" data-fx-ms="240">

短暫吃掉捲動輸入。它是上膛→擊發：進入視窗只上膛，下一次捲動才擊發，所以讀者停下來讀多久都不會把效果白白用掉。

afterimage殘影

<div data-fx="afterimage" data-fx-opacity=".035">

固定不動的內容複本，像上一幀沒有被擦乾淨。

eyelid睜眼

<section data-fx="eyelid" data-fx-slit="10" data-fx-open="70">

全黑一條縫，捲動後展開。用在角色醒過來、或者章節的第一口氣。

scramble亂序

<p data-fx="scramble" data-fx-level="0.25">

字序視覺打亂。DOM 裡永遠是正確原文——複製貼上、螢幕閱讀器、搜尋引擎拿到的都是原句。

stutter重複

<p data-fx="stutter" data-fx-times="3">

同一段遞增重複。用在一句話在腦子裡回音的時候。

怎麼用

  1. 把 effects/mycelium-fx.js 與 effects/mycelium-fx.css 複製到你的站。
  2. 在頁面引入它們。沒有 npm、沒有打包、沒有設定檔。
  3. 在要施法的段落標上 data-fx。
<link rel="stylesheet" href="mycelium-fx.css">
<script src="mycelium-fx.js"></script>

<p data-fx="freeze" data-fx-ms="240">門開的時候，他回頭看了一眼。</p>

另外還有一個全站開關：任何地方放一個 <input type="checkbox" data-fx-toggle>，讀者關掉之後全部效果立刻停用，並記在他自己的瀏覽器裡。

不可以破壞的四件事

  • 系統設成「減少動態」時，全部效果不啟用。
  • 鍵盤捲動與捲軸拖曳絕不攔截，只攔滑鼠滾輪與觸控。
  • 關掉 JavaScript 一樣讀得完整篇；效果全部是腳本後來加上去的。
  • scramble 的 DOM 永遠是正確原文，亂序只是視覺位移。

這個 repo 裡的另一半

mycelium skill——給終端機 agent（Claude Code、Codex、Gemini CLI…）用的長篇小說設定管理：角色與別名、關係、卷章狀態、伏筆逾期，全部在對話裡讀寫，資料放在你自己的 private repo。

它還有一個 mycelium graph 指令，把人物關係匯出成一個可以離線點開的 HTML。安裝與用法看 README。
